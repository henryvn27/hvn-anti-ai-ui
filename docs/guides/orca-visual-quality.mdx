# ORCA Visual Quality Guidance

This guide preserves the useful anti-generic UI standards that previously lived inside ORCA. ORCA should catalog this tool; Anti-AI UI should own the UI guidance and components.

## Goal

User-facing work should feel intentional, product-specific, and human-reviewed.

Aim for:

- clear hierarchy
- product-specific visual choices
- disciplined spacing and typography
- copy that sounds human, not generated
- loading, empty, and error states that preserve user confidence
- responsive behavior that keeps the same design idea across viewports

## Common AI-Looking Failures

- generic layout choices with no product point of view
- interchangeable color systems that could belong to any SaaS
- weak typography hierarchy
- decorative gradients or motion without product meaning
- repetitive section patterns
- bland stock phrasing
- too many ideas and no obvious visual priority

## Review Method

1. Lock the product goal and target user.
2. Choose visual direction from the product category and real content.
3. Define content hierarchy before styling.
4. Keep component language constrained.
5. Review screenshots or browser output, then refine.

## Artifact Spillover

Anti-generic quality applies beyond visible UI. Review release notes, loading states, empty states, onboarding copy, issue comments, and handoff summaries with the same standard.
